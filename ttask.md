# Техническое задание (ТЗ) на разработку мобильной CRM-системы учета посуды
## 1. Общие сведения и архитектура

* **Тип системы:** Микро-CRM (Web-приложение, адаптированное под Mobile-first / PWA).
* **Целевая платформа:** Смартфоны (iOS/Android через браузер).
* **Главный стек (рекомендованный для AI):** Python (FastAPI / Django) для Backend, PostgreSQL / SQLite для БД, HTML/Tailwind/JS или фреймворк для фронтенда.
* **Модель доступа:** Ролевая (Две роли: `Владелец/Админ` и `Продавец`).

---

## 2. Сущности и Структура Базы Данных (Data Models)

### Сущность `Товар (Product)`

* `id` (int, Primary Key)
* `name` (string) — Название посуды.
* `category` (string) — Категория (Казаны, Тарелки и т.д.).
* `cost_price` (float) — Себестоимость (Видит только Владелец).
* `wholesale_price` (float) — Оптовая цена (Видят оба).
* `stock_quantity` (int) — Текущий остаток на складе.
* `min_stock_level` (int) — Критический остаток для уведомлений.

### Сущность `Клиент (Client)` (Только для опта)

* `id` (int, Primary Key)
* `name` (string) — Имя/Название магазина.
* `phone` (string)
* `current_debt` (float, default: 0.0) — Текущая сумма долга клиента.

### Сущность `Сделка (Transaction)`

* `id` (int, Primary Key)
* `type` (enum: 'retail', 'wholesale') — Тип сделки.
* `client_id` (int, Optional, Foreign Key) — Для опта.
* `created_at` (timestamp, default: now)
* `total_amount` (float) — Общая сумма продажи.
* `owner_profit` (float) — Доля прибыли владельца.
* `seller_profit` (float) — Доля прибыли продавца (для розницы).
* `payment_status` (enum: 'cash', 'debt', 'partial_debt') — Статус оплаты.

### Сущность `Элемент Сделки (TransactionItem)` (Связующая для мульти-заказов)

* `id` (int, Primary key)
* `transaction_id` (int, Foreign Key)
* `product_id` (int, Foreign Key)
* `quantity` (int) — Количество проданного товара.
* `fact_price` (float) — Фактическая цена продажи за 1 шт (вводится вручную).

### Сущность `Расход (Expense)` (Только для Владельца)

* `id` (int, Primary Key)
* `amount` (float)
* `description` (string) — На что потрачено.
* `created_at` (timestamp, default: now)

## 4. Бизнес-Логика и Математические Формулы

### А. Сценарий «Розничная продажа» (Оформляет Продавец)

1. Продавец выбирает `Товар`, указывает `Количество (Q)` и вводит `Фактическую цену розницы (P_fact)` за единицу.
2. Система проверяет `Товар.wholesale_price (P_opt)`.
3. **Расчет прибыли по сделке:**
* $Общая\ прибыль = (P\_fact - P\_opt) \times Q$
* $Доля\ Продавца\ (40\%) = Общая\ прибыль \times 0.4$
* $Доля\ Владельца\ (60\%) = Общая\ прибыль \times 0.6$


4. Склад: `Товар.stock_quantity` уменьшается на $Q$.
5. Баланс продавца (виртуальный накопительный счет) увеличивается на его долю.

### Б. Сценарий «Оптовая продажа» (Оформляет только Владелец)

1. Владелец выбирает `Клиента`, добавляет **несколько товаров** (мульти-заказ). Для каждого указывает `Количество (Q)` и `Оптовою цену продажи (P_opt_fact)`.
2. **Расчет прибыли по сделке:**
* Для каждого товара в корзине: $Прибыль\ товара = (P\_opt\_fact - Товар.cost\_price) \times Q$
* $Общая\ прибыль = \sum (Прибыль\ товара)$
* Все 100% прибыли идут Владельцу.


3. **Учет оплаты:**
* Если выбран тип `Наличные`: Деньги идут в кассу дня.
* Если выбран тип `В долг`: Вся сумма (`total_amount`) плюсуется в `Клиент.current_debt`.
* Если `Частично`: Владелец вводит полученную сумму, остаток плюсуется в `Клиент.current_debt`.



### В. Сценарий «Погашение долга» (Оформляет только Владелец)

1. Владелец заходит в карточку клиента, нажимает «Погасить долг».
2. Вводит сумму внесения ($S$).
3. Система делает: `Клиент.current_debt = Клиент.current_debt - S`.
4. Сумма $S$ добавляется в отчет дня как «Погашение долга».

## 5. Требования к Интерфейсу (UI/UX) для AI-агента

### Интерфейс Продавца (Mobile-first):

* **Главный экран:** Две большие кнопки: «Новая продажа» и «Мой заработок за сегодня».
* **Форма продажи:** Выпадающий список товаров (с поиском) -> Поле ввода количества -> Поле ввода фактической цены. Кнопка «Сохранить». Показ сообщения: *«Успешно. Ваш заработок: Х сомони»*.

### Интерфейс Владельца (Дашборд / Итоги дня):

* **Складской блок:** Список товаров. Если `stock_quantity <= min_stock_level`, подсвечивать строку товара красным/желтым цветом с пометкой *«Заканчивается!»*.
* **Кнопка «Поставка»:** Форма для обновления остатков товара и ввода новой себестоимости.
* **Кнопка «Добавить расход»:** Поле ввода суммы и описания (например, "Обед 50", "Транспорт 100").
* **Ежедневный отчет (Финансы):** Экран, генерирующий данные за выбранный период (День/Неделя/Месяц):
* `Выручка Опт` = Сумма прибыли со всех оптовых накладных.
* `Выручка Розница` = Сумма долей 60% со всех розничных продаж.
* `Возвраты долгов` = Сумма принятых платежей по долгам за день.
* `Расход` = Сумма всех расходов за день.
* `Чистая Прибыль Владельца` = $(Выручка\ Опт + Выручка\ Розница + Возвраты\ долгов) - Расход$
* `Выплатить Продавцу` = Кнопка, показывающая текущий долг перед продавцом за его 40% доли. При нажатии обнуляет его счет (фиксация выдачи зарплаты).
